## 🔄 數據更新流程

### 初次加載
```
1. 用戶訪問 http://localhost:8000/
   ↓
2. 頁面加載 React 應用
   ↓
3. 自動查詢 /api/devices → 獲取設備配置
   ↓
4. 自動查詢 /api/power-latest → 獲取最新值
   ↓
5. 自動查詢 /api/power-history?device=X&days=7 → 獲取 7 天歷史
   ↓
6. Recharts 轉換數據並渲染圖表
   ↓
7. 頁面顯示完成
```

### 實時更新（每 10 秒）
```
1. 時間計時器觸發 (setInterval 10000ms)
   ↓
2. 自動查詢 /api/power-latest
   ↓
3. 更新所有卡片的「即時用電」數值
   ↓
4. 保持圖表不變（歷史數據）
```

### 進入群組詳細頁
```
1. 用戶點擊群組卡片
   ↓
2. URL 改變為 /#/group/高壓盤
   ↓
3. 頁面自動查詢該群組的所有設備
   ↓
4. 對每個設備查詢 /api/power-history?device=X&days=7
   ↓
5. 計算 7 日累計使用量 (kWh)
   ↓
6. 渲染群組詳細頁
```

## 📊 圖表類型和數據

### 全廠總用電圖表 (Factory Total Chart)
- **尺寸**：寬度 100%，高度 300px
- **數據**：全廠總用電 7 天歷史
- **時間軸**：自動格式化 (MM-DD HH:MM)
- **數值軸**：功率 (kW)
- **更新**：實時更新，但圖表歷史不變

### 群組卡片小圖表 (Group Card Mini Chart)
- **尺寸**：寬度 100%，高度 180px
- **數據**：該群組 7 天歷史總和
- **時間軸**：簡化標籤
- **數值軸**：功率 (kW)
- **目的**：快速預覽趨勢

### 群組總用電圖表 (Group Total Chart)
- **尺寸**：寬度 100%，高度 300px
- **數據**：該群組 7 天完整歷史
- **時間軸**：完整時間戳
- **數值軸**：功率 (kW)
- **精度**：最高精度

### 設備詳細圖表 (Device Chart)
- **尺寸**：寬度 100%，高度 200px
- **數據**：該設備 7 天完整歷史
- **時間軸**：完整時間戳
- **數值軸**：功率 (kW)
- **計算**：基於此數據計算 7 日累計 kWh

## 💡 使用提示

### 提示 1：快速導航
```
首頁 → 點擊「⚡ 電力監控儀表板」標題 → 回到首頁
      → 點擊任何群組卡片 → 進入詳細頁
詳細頁 → 點擊「[← 返回首頁]」按鈕 → 回到首頁
```

### 提示 2：理解 kWh 計算
```
能量消耗 (kWh) = 平均功率 (kW) × 時間 (h)

例如：
- 設備平均 100 kW，運行 24 小時
- 能量消耗 = 100 × 24 = 2,400 kWh

系統中：
- 採樣間隔：10 秒 = 0.00278 小時
- 每筆讀數貢獻能量 = 功率 × 0.00278 kWh
- 7 天累計 = Σ(每筆讀數的能量)
```

### 提示 3：實時更新
```
♓ 綠色點表示在線連接
✗ 紅色點表示離線狀態

如果看到「✗ 離線」：
1. 檢查後端伺服器是否運行
2. 檢查網絡連接
3. 刷新頁面
```

### 提示 4：時間範圍
```
系統硬固定 7 天時間範圍
- 首頁：顯示過去 7 天的全廠和群組數據
- 詳細頁：顯示過去 7 天的每個設備數據

未來版本將支援可選時間範圍選擇器
```

## 🔍 故障排除

### 問題：頁面無法加載
```
解決方案：
1. 檢查後端伺服器是否運行
   ps aux | grep main.py
2. 確認端口 8000 沒被佔用
   lsof -i :8000
3. 清除瀏覽器快取並刷新
   Ctrl+Shift+Delete (Chrome/Firefox)
```

### 問題：圖表無法顯示
```
解決方案：
1. 檢查瀏覽器控制台是否有錯誤
   F12 開發者工具 → Console
2. 確保 CDN 資源可訪問
   - React: unpkg.com/react@18
   - Recharts: unpkg.com/recharts@2.10.3
3. 檢查 API 是否返回數據
   curl http://localhost:8000/api/power-latest
```

### 問題：數據沒有更新
```
解決方案：
1. 檢查伺服器日誌
   tail -f /tmp/dashboard.log
2. 確保後端數據採樣正常
   curl http://localhost:8000/api/health
3. 檢查資料庫連接
   ls -la backend/power_data.db
```
